# ТЕХНИЧЕСКОЕ ЗАДАНИЕ - на разработку сайта интернет-магазина цветов BloomFlow

## 1. Общие сведения о проекте

Наименование проекта: BloomFlow

Тип проекта: многостраничный веб-сайт

Назначение сайта: предоставление пользователям возможности просматривать каталог букетов, выбирать товары, добавлять их в корзину, знакомиться с тарифами и переходить на страницы авторизации

## 2. Цель разработки сайта

Целью разработки является создание удобного и современного сайта интернет-магазина цветов, который позволит пользователям быстро находить подходящие букеты, добавлять их в корзину и взаимодействовать с основными разделами сайта.

## 3. Задачи сайта

Разрабатываемый сайт должен решать следующие задачи:
-  представление ассортимента цветочных композиций;
- удобная навигация между страницами;
- добавление товаров в корзину;
- удаление товаров из корзины;
- отображение общей стоимости заказа;
- сохранение выбранных товаров;
- предоставление информации о тарифах и подписках;
- переход на страницы входа и регистрации.

## 4. Целевая аудитория

Целевая аудитория сайта:
- пользователи, желающие приобрести цветы онлайн;
- клиенты, подбирающие букет для подарка;
- пользователи, заинтересованные в регулярной доставке цветов;
- молодая и взрослая аудитория, использующая интернет-магазины.

## 5. Структура сайта

Сайт включает следующие основные страницы:

* 5.1 Главная страница
Содержит основную информацию о магазине, приветственный блок, навигационное меню и переходы к основным разделам сайта.

* 5.2 Каталог
Страница с перечнем букетов и цветочных композиций. Пользователь может просматривать товары и добавлять их в корзину.

* 5.3 Корзина
Страница, на которой отображаются выбранные товары, их количество и итоговая стоимость. Также предусмотрена возможность удаления товаров.

* 5.4 Страница тарифов / подписки
Содержит информацию о тарифах, вариантах подписки или дополнительных предложениях.

* 5.5 Страница входа
Предназначена для авторизации пользователя.

* 5.6 Страница регистрации
Предназначена для создания новой учетной записи пользователя.

## 6. Функциональные требования

Сайт должен обеспечивать:
- отображение каталога товаров;
- добавление товаров в корзину;
- удаление товаров из корзины;
- подсчет общего количества товаров;
- подсчет итоговой стоимости заказа;
- сохранение корзины в localStorage;
- навигацию между страницами;
- наличие форм входа и регистрации.

## 7. Нефункциональные требования

К сайту предъявляются следующие нефункциональные требования:
- корректная работа в современных браузерах;
- удобный и понятный интерфейс;
- эстетичный дизайн, соответствующий тематике цветочного магазина;
- логичная структура страниц;
- читаемый и структурированный программный код.

## 8. Технологии разработки

При разработке сайта используются следующие технологии:
- HTML5;
- CSS3;
- JavaScript;
- localStorage для хранения данных корзины.

## 9. Требования к дизайну

Дизайн сайта должен:
- соответствовать тематике цветочного магазина;
- быть современным, легким и визуально привлекательным;
- содержать удобные элементы навигации;
- обеспечивать комфортное восприятие информации;
- выделять карточки товаров и основные кнопки взаимодействия.

## 10. Ожидаемый результат

В результате выполнения проекта должен быть разработан многостраничный сайт интернет-магазина цветов, который:
- содержит каталог товаров;
- позволяет работать с корзиной;
- сохраняет выбранные товары;
- имеет страницы входа и регистрации;
- обладает понятной структурой и привлекательным внешним видом.

## 11. Перспективы развития проекта

В дальнейшем проект может быть дополнен следующими возможностями:
- поиском по каталогу;
- фильтрацией товаров;
- оформлением заказа;
- подключением серверной части;
- реализацией полноценной авторизации;
- адаптацией под мобильные устройства;
- интеграцией платежной системы.
